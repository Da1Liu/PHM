技术路线验证 · 通俗讲解版

很多传感器一起，
能不能学会"机器健康长什么样"？

这份文档在做一件事：验证"把振动、温度、电流、压力等多个传感器合起来，建立一个'健康状态'的统计模型，再用它发现异常"这条路到底走不走得通。下面把它从头讲清楚。

↓ 往下滑，我们一步一步来

?先搞清楚：到底要解决什么问题

为什么不能"每个传感器单独设个报警线"就完事了。

打个比方 · 体检

你去体检，每一项指标——血压、心率、血糖——都在正常范围内，单看没毛病。但医生可能仍然皱眉：因为你"血糖偏高 + 血压偏高 + 体重偏高"同时出现，这个组合本身就不正常，哪怕每一项都没越线。

机器也一样。一台主轴：振动、温度、电流可能各自都在"正常区间"，但健康的机器里它们之间有固定的配合关系（转得快→电流大→温度升）。如果某天"电流没怎么变，温度却异常高"，说明这个配合关系坏了——这往往是故障的早期信号。

❌ 传统做法：单通道阈值

给每个传感器画一条红线，超了就报警。简单，但只看每个变量自己，完全看不见"变量之间关系变了"这种异常。早期、轻微的联合退化容易漏掉。

✅ 本文路线：联合健康基线

把多个传感器拼成一个向量，学习健康时它们的联合分布（不仅每个变量的范围，还有变量之间的相关关系）。新数据离这个"健康形状"越远，越可疑。

一句话目标：验证"多传感器 + 协方差矩阵"能不能学出一个稳定的'健康长什么样'，并且比单通道阈值更早、更准地发现异常。

①核心直觉：什么是"协方差 / 马氏距离"

这是整条路线的灵魂。用一张图你就懂了——别怕公式。

想象只有 两个传感器：横轴 = 振动，纵轴 = 温度。把健康时收集到的很多个样本点画出来，它们不会乱成一团，而是形成一个有方向的"云"（因为振动大时温度往往也高，二者相关）。

健康样本（蓝色云）
椭圆 = 健康的"形状"范围
被检测的可疑点

蓝色点形成一个斜着的椭圆：这就是健康状态的"形状"。绿色椭圆是它的边界。

"马氏距离" 在量什么

普通距离是"离中心几厘米"。马氏距离是"顺着这朵云的形状，离中心几个标准差"。同样的直线距离，顺着椭圆长轴方向算很近，横穿短轴方向算很远。它自动考虑了"哪些方向本来就该一起变"。

公式 D² = (x−μ)ᵀ Σ⁻¹ (x−μ) 里那个 Σ（协方差矩阵）就是"云的形状"，μ 是云的中心。

关系异常 = 这条路线的杀手锏 ⭐

点击上面的 "关系异常" 按钮：那个红点的振动、温度各自都落在正常范围里（投影到横轴、纵轴都不超线），但它落在椭圆外面——因为它违反了"振动高温度也该高"的配合关系。

单通道阈值看不见它，协方差能看见。这正是体检比喻里"每项都正常但组合异常"的情况。

没有现成的主轴数据，先借一套公开的"液压系统"数据来证明这条数学路线成立。

③验证分三步走（文档里的"停点"）

整个验证不是一锤子，而是层层加码，每步都设了一个"该不该继续"的检查点。

停点 1 · 自检
关系真的存在吗？

提取特征，看传感器之间到底有没有相关性，矩阵会不会"病态"。

→
停点 2 · 对照
协方差比单通道强吗？

真实泄漏故障上，7 个模型 PK。

→
停点 3 · 关系异常
能抓住"关系坏了"吗？

人造"边际正常但配合乱掉"的样本，看谁抓得住。

停点 1 发现了什么

指标值大白话
跨传感器平均相关强度0.331明显不为零 → 传感器之间确实有配合关系 ✅
最强一对相关0.996有两个传感器几乎是"复制粘贴"，信息高度重叠
协方差矩阵最大特征值5.96某个方向上数据变化幅度很大
协方差矩阵最小特征值0.000000053另一个方向几乎不变（因为有高度相关的传感器对）
条件数 = 最大/最小1.12 亿矩阵严重病态，直接求逆时微小误差会被放大上亿倍 ⚠️
为什么 1.12 亿是个大问题？
求逆时需要用"最小特征值的倒数"，0.000000053 的倒数是 1880 万。任何测量噪声、浮点误差都会被疯狂放大。一般条件数 > 100 万就很危险，1 亿量级意味着裸矩阵求逆的结果基本不可信。这就是为什么必须用正则化（加稳定剂）或 PCA（丢掉病态方向）。

结论：关系存在（好消息），但 14 维裸协方差矩阵太病态，工程上不能直接用，要靠"正则化"或"PCA 降维"来稳住。

④参赛的 7 个"选手"（对照模型 M0–M6）

为了证明"完整协方差有用"，必须和一堆更简单/更朴素的方法比。把它们想成赛跑选手。

M0 · 单变量阈值 最朴素

每个传感器各看各的，取最离谱的那个当分数。代表传统单通道报警。

M1 · 对角协方差 半成品

考虑了每个变量的"胖瘦"，但假装变量之间没关系。相当于"各算各的再加起来"。

M2 · 完整协方差 主角

用上全部变量之间的关系。理论上最强，但前面说了，它的矩阵太病态，工程上有风险。

M3 · 正则化协方差 工程候选

给病态矩阵"加点稳定剂"，既用上关系又不容易翻车。是更现实的版本。

M4 · PCA + T²

先降维，抓住几个最主要的"共同变化模式"，再在这个低维空间里量距离。

M5 · PCA + T² + SPE 推荐

在 M4 基础上再加一个 SPE（残差）：专门盯"偏离主模式的部分"。一会儿你会看到，SPE 才是真正的主力。

M6 · 打乱相关性（陷阱对照）

故意把训练数据每列独立洗牌，破坏真实关系再建协方差。如果它也很强，说明前面的强是碰运气而非真关系。这是个"照妖镜"。

⑤怎么读这些数字（指标词典）

结果表里会反复出现这几个词，先把它们翻译成人话。

AUC 0 ~ 1
把健康和故障样本分开的能力。随便瞎猜=0.5，完美区分=1.0。
越接近 1 越好
Spearman −1 ~ 1
异常分数是否随"泄漏越严重"而单调升高。越高说明越能反映严重程度，不只是有无。
正且越高越好
FAR（误报率）%
健康样本里被错误报警的比例。报警别太爱"狼来了"。
越低越好（但见"诚实的边界"）
条件数 condition number
矩阵"病态"程度。太大 → 求逆不稳，结果像在悬崖边走。
越小越稳
offdiag 相关强度
传感器之间平均有多相关。接近 0 = 各管各的，协方差没用武之地。
明显非零才有意义
T² vs SPE
T²：在"主要模式"内部的异常。SPE：偏离主要模式的"残差"异常。两者互补。
关系异常常靠 SPE 抓住

⑥实际结果：三步分别看到了什么

这里是验证的核心证据。我们一组一组看。

结果 A · 真实泄漏太"简单"了

用上全部 14 个特征时，所有模型 AUC 都 = 1.000，全部满分。

满分听起来很棒，但其实是个问题：因为有几个单变量（如流量、流量压力比）本身就极其好分，用不用协方差都一样满分，根本比不出高下。所以下一步要故意把这些"作弊特征"删掉，逼出真本事。

结果 B · 删掉"作弊特征"后 —— 这才是关键证据 ⭐

把单变量就极易区分的强特征剔除，剩下 10 个"弱"特征。再看谁还行：

✅ 删掉强特征后，协方差类（正则化 0.98）和 PCA+SPE（满分）依旧明显甩开对角协方差（0.75）、单变量（0.68）和"打乱对照"（0.71）。这说明收益来自传感器之间真实的关系，不是碰运气。这是支撑整条路线的最硬的证据。

结果 C · 关系异常测试 —— 杀手锏场景

什么叫"人造样本"？
假设健康校准集有 68 条样本，每条是 10 维向量（10 个传感器）。代码做了这个操作（以"全特征打乱"为例）：

for 每个传感器列: 把这一列的 68 个值独立洗牌
结果：生成 68 条"伪异常"样本，每个传感器的数值仍然从健康样本里抽（所以单看任何一个传感器，数值都在正常范围），但传感器之间的配对关系被打乱了。

举例：原本样本 1 是 [ps1=150, eps1=2400, ts2=40]，样本 2 是 [ps1=151, eps1=2410, ts2=41]。打乱后可能变成伪异常样本 [ps1=样本23的, eps1=样本7的, ts2=样本55的]——每个值都健康，但"ps1 高时 eps1 也高"的配合关系被破坏了。
AUC 怎么算的？有什么价值？
把 68 条真健康样本（标签=0）和 68 条伪异常样本（标签=1）混在一起，让模型打分，算 AUC。AUC 量的是：模型能否区分"真健康"和"关系被打乱的伪异常"。

价值：验证模型是否具备捕捉"关系异常"的能力。真实的泵泄漏会同时改变流量、压力、功率的绝对值，所以单变量也能抓（太简单）。但有些早期故障可能是"摩擦增加一点点、热膨胀增加一点点、振动频率偏移一点点"，每项都没越线但组合不对——这个测试就是在模拟这种场景，看模型有没有这个能力。

为什么单变量/对角协方差 AUC≈0.5（瞎猜）？
这不是因为它们能力差，而是数学设计上注定的：它们只看每个传感器自己的数值。而伪异常样本的每个传感器数值都是从健康样本里拿的，在它们眼里和真健康样本完全一样。所以 AUC≈0.5 是必然。

协方差和 PCA+SPE 能抓住，说明它们真的学到了"ps1 高时 eps1 也该高"这种关系，打乱后能识破。

最后一行"仅功率打乱"为什么大家都抓不住？ 说明在这个数据里，功率(EPS1)与其它传感器之间的线性协方差耦合相对独立或较弱，打乱它不会显著违反模型学到的健康结构。类比：乐队里打乱三角铁演奏者的时机，基本听不出——不是他/她贡献大，而是和其它乐器的协同关系弱。

⚑诚实的边界：哪些结论要打折扣

一个负责任的验证，必须说清自己没证明什么。这几点很重要，门外汉尤其要知道，免得被"满分""完胜"误导。

1. "误报率 1.47%" 是个数学巧合，不能当真。
代码里用同一批健康数据既定阈值又算误报——取"99 分位"当线，那必然只有约 1% 的点超线。68 个点里超 1 个 = 1.47%。所有模型都恰好是这个数，因为它跟模型好坏无关，是被算法定义"逼出来"的。真要评估误报，需要一批没参与定阈值的独立健康数据。
2. 关系异常测试"完胜"，有一半是设计决定的。
那个测试只打乱关系、不改每个变量自己的数值。而单变量/对角模型本来就只看每个变量自己——它们在数学上注定看不见这种异常，AUC≈0.5 是必然，不是"实测发现"。

所以它证明的是"协方差有能力抓关系异常"（必要条件），不等于"真实主轴故障一定是这种'边际正常只是关系乱'的形态"。真实故障往往会同时改变幅值和关系，那时候单变量阈值也能抓到一部分。

它的价值在于：如果真机上确实出现"每项都在正常范围但组合不对"的早期退化，这个测试证明协方差/PCA 具备识别能力，而单通道不具备。至于真实故障有没有这个特征，需要在真机数据上验证。
3. 立功的其实是 SPE（残差），不是 T²（主成分）。
删掉强特征后，只用 T² 的 M4 只有 0.64，加上 SPE 的 M5 直接 1.0。也就是说，真正抓住异常的是"偏离主模式的残差"部分，而不是"主要共变结构"本身。所以工程落地时 SPE 这一项不能省，光靠 PCA 主成分是不够的。
这些不是要否定路线，而是把它的"证明力"摆正：
→ 路线可行这一条，靠的是 结果 B（删掉作弊特征后协方差仍胜出，且是合法的样本外检验）。
→ 误报率、关系异常的"现实意义"，需要真机数据 + 独立测试集才能进一步坐实。

✓最终结论：这条路到底成不成立

文档给了三档判定。结合证据，目前落在"成立，但要用对版本"。

✅ 强通过

  • 有明确跨传感器关系
  • 正则化/PCA 优于对角
  • 误报率 < 5%
  • 分数随严重度升高
  • 关系异常上完胜

⭐ 实际落点：弱通过

  • 真实故障上各法接近
  • 但删强特征后协方差明显更优
  • 协方差需正则化/PCA 才稳
  • 结论：路线可行，工程上用正则化协方差或PCA+T²+SPE

✗ 不通过

  • 关系很弱
  • 协方差/PCA 都不比对角强
  • 关系异常也没优势
  • → 需换数据集
