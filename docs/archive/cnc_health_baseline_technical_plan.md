## 1. 问题边界与核心约束

### 1.1 场景定义

```
设备类型：重型机床（进给系统 / 主轴系统 / 液压系统）
采集窗口：热机后空运行阶段（标准热机NC程序执行期间）
监测目标：设备健康状态的缓慢退化趋势
输出形式：每次空运行产出一个健康度值 [0, 1]
```

### 1.2 约束清单

| 约束 | 说明 |
|------|------|
| 异常数据极度稀缺 | 本质是 One-Class 问题，不能依赖异常标签训练 |
| 工况组合繁多 | 空运行消除刀具/工艺/材料变量，仅剩转速×进给速度两个维度 |
| 不做热变形研究 | 温度只作协变量统计处理，不建热误差模型 |
| 厂内无法预采集 | 分拆发货，现场重新装配，必须在线自动建立基线 |
| 用户不停机 | 仅交班时有采集窗口，每天 2～3 次 |
| 不侵入CNC总线 | 不读取坐标反馈信号，不接入总线 |
| 边缘侧资源有限 | 特征必须是标量，算法轻量，无FFT，无矩阵训练 |
| 第一阶段只做异常检测 | 不做故障定位，不做空间频率分析 |

### 1.3 排除项

- 热变形补偿、热误差建模
- 切削过程中的监测（仅空运行）
- 故障类型识别和部位定位（第二阶段）
- 高精度坐标对齐分析
- 深度学习模型（数据量不足）

## 2. 背景知识

### 2.1 为什么选择空运行阶段

空运行阶段消除了切削力、刀具磨损、工件材料等强干扰变量，使得传感器信号的变化主要来自机械系统本身的健康状态变化，而非工艺参数差异。这是在工况多变的机床上实现可比较健康度的关键前提。

### 2.2 One-Class 问题的本质

设备健康监测在无异常标签时属于 One-Class Classification 问题：只学习"正常状态长什么样"，任何偏离正常的状态都被判定为异常。主要方法有三类：

**统计过程控制（SPC）类**：Hotelling T²、CUSUM、EWMA 控制图。需要正常样本建立参考分布，计算当前样本到正常分布中心的统计距离。可解释性强，适合小数据场景。

**基于密度估计类**：Isolation Forest、One-Class SVM、LOF。对数据分布假设更弱，适合非高斯分布特征。计算量略高，可解释性较弱。

**深度学习类**：Autoencoder 重构误差、Deep SVDD、Transformer 异常分数。特征提取能力强，但需要大量正常样本，冷启动困难，不适合第一阶段。

本方案选择 SPC 类，原因是：冷启动样本有限、需要可解释性向用户说明原因、边缘侧资源有限。

### 2.3 PCA + Hotelling T² 原理

**PCA（主成分分析）** 的作用是处理特征间的相关性。当多个特征高度相关时（如振动 RMS 和电流 RMS），直接计算多维距离会重复计数相关特征的贡献。PCA 通过坐标旋转找到数据中真正独立的变化方向（主成分），使得每个主成分之间互不相关。

数学上，PCA 对基线期特征矩阵 X（形状 n×p）进行 SVD 分解：

```
X = U · S · V^T
W = V[:, :k]   # 保留前k个主成分方向，形状 p×k
```

**Hotelling T²** 在 PCA 投影后的空间里计算当前样本到正常分布中心的统计距离：

```
z = W^T · x_norm          # 投影到主成分空间，p维→k维
T² = z^T · Λ⁻¹ · z        # Λ 是基线期各主成分的方差（对角矩阵）
```

T² 服从 F 分布，因此可以精确计算置信区间：

```
UCL = k(n²-1) / (n(n-k)) · F(α; k, n-k)   # 理论UCL
```

实践中通常直接用基线期 T² 值的 99% 分位数作为 UCL，更鲁棒。

**健康度映射**：

```
health = exp(-α · T² / UCL)
```

T²=0 时 health=1；T²=UCL 时 health≈exp(-α)；α 通常取 3，使 UCL 对应 health≈0.05。

### 2.4 EWMA 趋势监测原理

指数加权移动平均（EWMA）对健康度时间序列做平滑，抑制单次采集的随机噪声，突出缓慢趋势：

```
EWMA_n = λ · health_n + (1-λ) · EWMA_{n-1}
```

λ 是平滑参数，通常取 0.1～0.2。λ 越小，对历史值权重越高，曲线越平滑，但对突变响应越慢。λ 越大，对新数据响应越快，但噪声抑制效果越差。

UCL（上控制限）设为基线期 EWMA 序列的 3σ，表示正常波动范围的上界。

### 2.5 协变量回归去除原理

温度影响特征基线但不直接反映退化，属于混淆变量（Confounder）。若不处理，夏季健康度系统性偏低、冬季偏高，产生季节性虚假告警。

处理方法：在基线期对每个特征拟合线性回归模型：

```
feature = α · T_ambient + β · T_motor + γ · T_spindle + δ · T_oil + ε
```

后续采集的特征值减去回归预测值，得到残差：

```
x_residual = x_raw - (α·T + β·T_motor + ...)
```

残差代表去除温度影响后的纯机械健康信息，进入后续建模。

### 2.6 磨合期特征

新机床装配完成后，轴承滚道、丝杠滚珠、导轨等摩擦副处于初始磨合阶段。磨合期特征表现为：振动 RMS 和电流 RMS 在前数十至数百小时内单调下降，直至趋于稳定。

磨合期数据不应用于建立基线，否则基线描述的是"磨合期偏高状态"，磨合完成后特征降低，系统会误判健康度上升，掩盖真实退化信号。

本方案用**稳定性加权**替代"磨合结束时刻"判定：每条样本根据其与邻居的相似程度计算权重，不稳定的磨合期样本自动获得低权重，不需要人工标定磨合终点。

## 3. 整体技术路线

```
交班触发标准热机NC程序
        ↓
【采集层】
  进给系统：伺服电流 + 振动
  主轴系统：振动（径向双向）+ 主轴电流
  液压系统：压力 + 流量 + 油温
        ↓
【预处理层】
  数据质检（热机程序ID / 上班时长 / 异常值剔除）
  工况标准化（转速×进给 硬分层 + 温度线性回归去除）
  污染检测（Isolation Forest，退化样本降权）
        ↓
【特征层】
  全部为标量，时域统计量
  进给系统 6个 + 主轴系统 4个 + 液压系统 4个 = 14个特征
        ↓
【建模层（渐进切换）】
  n < 30    工程先验阈值 + CUSUM相对变化检测
  30~100    分位数评分（各特征独立）
  > 100条跨日  PCA + Hotelling T²
        ↓
【输出层】
  健康度 [0,1] + EWMA趋势 + 置信度标注
```

## 4. 数据采集设计

### 4.1 热机NC程序结构

```
阶段一：暖机段（不采数据）
  各轴低速往复 5~10 分钟
  目的：达到热平衡状态
  不采集任何用于健康度计算的数据

阶段二：采集段（唯一的采集段）
  固定速度 v_std（建议快移速度的 60%）
  固定行程范围（全行程或指定安全区间）
  往复 15~20 次
  固定加减速参数（时间常数锁定，不允许自适应调整）
  全程连续采集电流 + 振动 + 压力/流量
```

**NC程序固定的参数**：换向速度、加减速时间常数、行程起终点坐标、往复次数。这些参数一旦确定不再变动，大修后才重新标定。

### 4.2 采集段的信号截断

加减速段速度不恒定，对部分特征的提取有影响。特征提取时截断加减速段，保留匀速段：

```
判断方法（不依赖坐标）：
  电流从静止到运动有明显跃升（加速段）
  电流从运动到静止有明显下降（减速段）
  跃升结束到下降开始之间的平稳段 = 匀速段

匀速段长度 = 行程 / 速度 - 2×加速时间
建议匀速段占总行程的 70% 以上
```

### 4.3 交班数据质检规则

| 检查项 | 通过条件 | 不通过处理 |
|--------|---------|-----------|
| 热机程序ID | 与标准程序一致 | 丢弃该次数据 |
| 上班时长 | ≥ 4 小时 | 降权使用，单独分组 |
| 环境温度记录 | 温度传感器有效读数 | 有限使用，标注缺失 |
| 异常值检测 | 信号无饱和、无断线 | 丢弃该次数据 |

**上班时长的重要性**：热机前的上班时长决定了机床热平衡状态。短班次（<4小时）和长班次（>8小时）的热机采集特征不可比，必须分组或降权处理。

### 4.4 工况分层策略

```
硬分层（离散档位，工程语义明确）：
  主轴转速档：按NC程序设定转速分组
  进给速度档：按NC程序设定进给分组
  班次时长分箱：< 4h / 4~8h / > 8h

软回归（连续处理，不损失样本）：
  温度协变量：环境温度、电机温度、主轴轴承温度、油温
  对每个特征分别拟合线性回归，提取残差
```

## 5. 特征体系

### 5.1 进给系统（6个特征）

| 特征名 | 信号来源 | 物理含义 | 退化机制 | 计算量 |
|--------|---------|---------|---------|--------|
| 电流 RMS | 伺服电流（匀速段） | 整体摩擦水平 | 丝杠预紧力变化、润滑失效时升高 | 极轻 |
| 电流标准差 | 伺服电流（匀速段） | 摩擦力均匀性 | 丝杠局部磨损导致摩擦力不均时升高 | 极轻 |
| 正反向电流均值差 | 伺服电流（匀速段，按换向时刻分段） | 方向性退化 | 反向间隙增大或单侧导轨磨损时偏移 | 极轻 |
| 振动 RMS | 振动加速度（全段） | 整体振动水平 | 轴承滚道磨损、导轨间隙增大时升高 | 极轻 |
| 峭度（Kurtosis） | 振动加速度（全段） | 冲击敏感指标 | 早期轴承剥落产生冲击脉冲，峭度先于RMS升高 | 轻量 |
| 往复间RMS标准差 | 本次热机N次往复的各次电流RMS | 运动一致性 | 反映本次运行内部的重复性，冷启动即可用 | 极轻 |

**协变量**：环境温度 T_ambient（影响润滑脂粘度→摩擦）、电机温度 T_motor（影响线圈电阻→电流基线）。

**峭度的特殊性**：健康轴承峭度约为 3（正态分布理论值）。峭度升高（>6）表示存在冲击，是早期故障指标；峭度降低至接近 3 但振动 RMS 同时升高，可能意味着缺陷已从点状扩展为片状，反而是更严重的退化阶段。因此峭度需与 RMS 联合解读。

### 5.2 主轴系统（4个特征）

| 特征名 | 信号来源 | 物理含义 | 退化机制 | 计算量 |
|--------|---------|---------|---------|--------|
| 合振幅值 RMS | 径向 X + Y 振动：sqrt(X²+Y²) 的 RMS | 径向振动总量 | 轴承整体磨损时升高，消除安装方向差异 | 极轻 |
| 振动峭度 | 合振信号 | 冲击敏感指标 | 主轴轴承早期剥落的最早期信号 | 轻量 |
| 主轴电流 RMS | 主轴驱动电流 | 空载转矩水平 | 轴承摩擦增大、动平衡变差时升高 | 极轻 |
| 振动 RMS 时间标准差 | 采集窗口内振动的稳定性 | 热机稳定性 | 主轴热机过程中振动应趋稳，持续不稳说明状态异常 | 极轻 |

**协变量**：主轴前轴承座温度 T_front、后轴承座温度 T_rear。仅用于回归去除对振动和电流基线的影响，不建立热变形模型，不预测热误差。

**多传感器融合时机**：
- 冷启动期：X/Y 合并为合振幅值（单特征，基线稳定）
- PCA+T² 阶段：X/Y 可拆开输入，让模型学习两个方向的相关结构，能捕捉方向性故障（不平衡等）

### 5.3 液压系统（4个特征）

| 特征名 | 信号来源 | 物理含义 | 退化机制 | 计算量 |
|--------|---------|---------|---------|--------|
| 稳态压力均值 | 压力传感器（建压后稳定段） | 系统密封状态 | 溢流阀磨损或管路泄漏时下降 | 极轻 |
| 稳态压力标准差 | 压力传感器（建压后稳定段） | 压力稳定性 | 蓄能器气囊破裂或阀件磨损时升高 | 极轻 |
| 流量压力比 Q/P | 流量均值 / 压力均值 | 泵容积效率代理 | 内泄漏增大→同等压力需更多流量→Q/P 升高 | 极轻 |
| 压力建立时间 | 压力时间序列 | 泵和阀的响应特性 | 泵输出下降或阀响应变慢时延长 | 极轻 |

**协变量**：液压油温 T_oil。油温对液压特征影响最显著，粘度随温度变化可达 50% 以上，直接影响 Q/P 基线。若不回归去除，夏冬季节 Q/P 差异可能超过真实退化信号幅度。

**流量压力比的设计原理**：单独使用流量或压力都受负载工况影响；流量与压力的比值在不同工况点都稳定反映泵的内泄漏状态，是工况无关的效率指标。

### 5.4 特征汇总

```
进给系统：6 个标量特征
主轴系统：4 个标量特征
液压系统：4 个标量特征
协变量：  4 个（环境温度、电机温度、主轴轴承座温度、油温）

合计：14 个健康特征（全部为时域标量，无需FFT，无需坐标对齐）
      4 个协变量（线性回归去除，不进入健康度模型）
```

## 6. 建模层：渐进式健康度算法

### 6.1 三阶段渐进切换

系统从第一次空运行起就能输出健康度，随数据积累自动升级算法，对外接口不变。

#### 阶段一：n < 30 条（工程先验期）

**算法**：工程阈值规则 + CUSUM 相对变化检测

```python
# 健康度来自两部分：
# 1. 物理极限阈值检查（来自电机铭牌、轴承手册）
def check_physical_limits(features, thresholds):
    violations = sum(1 for f, (lo, hi) in zip(features, thresholds)
                     if not lo <= f <= hi)
    return 1.0 - violations / len(features)

# 2. CUSUM 检测相对于初始值的漂移
def cusum_health(x, x0, sigma0, k=0.5, h=5):
    # k：允许漂移量（sigma的倍数）
    # h：报警阈值
    s_plus  = max(0, s_plus_prev  + (x - x0)/sigma0 - k)
    s_minus = max(0, s_minus_prev - (x - x0)/sigma0 - k)
    anomaly = (s_plus > h) or (s_minus > h)
    return 0.2 if anomaly else 1.0
```

**置信度标注**：灰色显示，标注"基线建立中（n=xx）"

#### 阶段二：30～100 条跨日（分位数评分期）

**算法**：各特征独立分位数评分，加权平均

分位数评分的映射规则：
- 当前值 ≤ P80：该特征健康度 = 1.0（正常范围）
- P80 < 当前值 ≤ P95：线性下降，1.0 → 0.0
- 当前值 > P95：健康度 = 0.0

```python
def percentile_score(x, p80, p95):
    if x <= p80:
        return 1.0
    elif x <= p95:
        return 1.0 - (x - p80) / (p95 - p80)
    else:
        return 0.0

# 综合健康度：加权平均
weights = {
    'current_rms': 0.20,
    'current_std': 0.10,
    'asym_current': 0.10,
    'vib_rms': 0.15,
    'kurtosis': 0.15,
    'pass_consistency': 0.10,
    # ...其余特征
}
health = sum(percentile_score(features[k], p80[k], p95[k]) * w
             for k, w in weights.items())
```

**注意**：分位数评分把各特征视为独立特征，忽略特征间相关性，高度相关的特征（如振动 RMS 和电流 RMS）会重复计数同一退化模式。这是阶段二的已知局限，阶段三通过 PCA 解决。

**置信度标注**：正常显示，标注"基线积累中（n=xx）"

#### 阶段三：> 100 条且跨日 ≥ 14 天（PCA + Hotelling T² 期）

**为什么跨日天数比样本数更重要**：跨日天数反映了温度和状态的多样性覆盖，100条数据若在3天内采集，协方差矩阵描述的仍是单一热平衡状态，而非设备正常变化范围。

**训练阶段（云端离线执行）**：

```python
from sklearn.preprocessing import StandardScaler
from sklearn.decomposition import PCA
import numpy as np

# 1. 基线期特征矩阵（形状：n×14）
X_baseline = ...  # 清洁样本，已去除温度影响

# 2. 标准化
scaler = StandardScaler()
X_norm = scaler.fit_transform(X_baseline)

# 3. PCA：选择保留95%方差的主成分数
pca = PCA(n_components=0.95)
pca.fit(X_norm)
W = pca.components_.T   # 形状：14×k
Lambda = pca.explained_variance_  # 各主成分方差，形状：(k,)

# 4. 计算基线期每条样本的T²
Z_baseline = X_norm @ W
T2_baseline = np.array([z @ np.diag(1/Lambda) @ z for z in Z_baseline])

# 5. UCL：99%分位数（也可用F分布理论值）
UCL = np.percentile(T2_baseline, 99)

# 下发到边缘侧：W, Lambda, UCL, scaler参数, 回归系数
```

**推理阶段（边缘侧在线执行，每次热机后运行）**：

```python
# 边缘侧存储：W(14×k), Lambda(k,), UCL, baseline_mean(14,), baseline_std(14,), reg_coefs
# 内存占用：全部参数 < 5KB

def compute_health(x_raw, temperatures, model_params):
    W, Lambda, UCL = model_params['W'], model_params['Lambda'], model_params['UCL']
    reg_coefs = model_params['reg_coefs']
    mean, std = model_params['baseline_mean'], model_params['baseline_std']

    # 1. 去除温度影响
    x_residual = x_raw - reg_coefs @ temperatures

    # 2. 标准化
    x_norm = (x_residual - mean) / std

    # 3. PCA 投影
    z = W.T @ x_norm

    # 4. Hotelling T²
    T2 = z @ np.diag(1 / Lambda) @ z

    # 5. 健康度映射
    health = np.exp(-3 * T2 / UCL)

    return health, T2
```

**置信度标注**：正常显示，无额外标注。

### 6.2 阶段过渡期加权混合

为避免算法切换时刻的健康度跳变，50～200 条样本区间内两套算法加权混合：

```python
def blended_health(n_samples, health_percentile, health_T2):
    if n_samples < 50:
        return health_percentile  # 纯分位数
    elif n_samples <= 200:
        w_T2 = (n_samples - 50) / 150  # 线性过渡
        return (1 - w_T2) * health_percentile + w_T2 * health_T2
    else:
        return health_T2  # 纯 T²
```

### 6.3 EWMA 趋势输出

```python
lambda_ewma = 0.15  # 平滑参数

ewma = lambda_ewma * health + (1 - lambda_ewma) * ewma_prev

# UCL：基线期EWMA序列的3σ
ucl_ewma = ewma_baseline_mean + 3 * ewma_baseline_std
```

### 6.4 T² 特征贡献解释（阶段三）

当 T² 超过 UCL 触发告警时，可将 T² 分解到各主成分的贡献，再映射回原始特征，帮助判断哪个特征异常：

```python
# 各主成分对T²的贡献
component_contributions = z**2 / Lambda  # 形状：(k,)

# 映射回原始特征空间（近似贡献，用于说明方向）
feature_contributions = np.abs(W) @ component_contributions  # 形状：(14,)
```

## 7. 基线建立策略

### 7.1 新机床：自动在线建基线

新机床无厂内预采集数据，采用以下策略：

**污染检测**（防止退化样本或磨合期样本污染基线）：

```python
from sklearn.ensemble import IsolationForest

# 每积累10条新数据，重新评估一次样本池
clf = IsolationForest(contamination=0.05, random_state=42)
scores = clf.fit_predict(baseline_pool)

# 移除得分为 -1 的可疑样本（保留记录供人工审核）
clean_pool = baseline_pool[scores == 1]
```

**稳定性加权**（替代"磨合结束时刻"判定）：

每条样本根据其与最近邻样本的距离计算权重，不稳定的磨合期样本自动获得低权重：

```python
from sklearn.neighbors import NearestNeighbors

def compute_stability_weight(X, n_neighbors=5, sigma=1.0):
    nbrs = NearestNeighbors(n_neighbors=n_neighbors).fit(X)
    distances, _ = nbrs.kneighbors(X)
    avg_dist = distances[:, 1:].mean(axis=1)  # 排除自身
    weights = np.exp(-avg_dist / sigma)
    return weights / weights.sum()
```

**时间线估算**（每天2～3次交班，每次产生1条有效样本）：

```
装机后第 1~2 周   稳定性加权积累，工程先验阈值上线，CUSUM 监测相对变化
装机后第 3~4 周   30~50 条跨日样本，分位数基线成型
装机后第 6~8 周   100 条以上跨日14天，PCA+T² 上线
```

### 7.2 同型号机床迁移策略

**可迁移（结构知识）**：
- 物理极限阈值（来自设备规格书，与个体无关）
- 特征有效性知识（哪些传感器对哪类退化敏感）
- 温度协变量响应方向（回归系数的符号和粗略量级）
- 磨合期曲线形状（用于辅助判断磨合阶段，不用于绝对值比较）

**不可迁移（数值参数）**：
- 特征绝对均值（电机型号不同，空载电流均值可能差 20% 以上）
- 协方差矩阵（轴承型号和装配精度不同，特征间相关结构不同）
- 振动幅值基线（安装水平度、地基刚性差异可导致同型号机床振动相差数倍）

### 7.3 存量机床：四象限策略

按"有无维护记录"和"有无同型号新机基线"分四种情况：

| | 有同型号新机基线 | 无同型号新机基线 |
|---|---|---|
| **有维护记录** | 最优：新机基线做参考上界，大修后数据建当前基线，健康度相对新机可量化 | 次优：维护记录划分健康段，大修后首批数据建基线，物理极限做兜底阈值 |
| **无维护记录** | 可用：新机基线做结构先验，当前数据估计偏移量，标注"存量模式" | 最难：只能建"当前状态基线"，监测未来相对变化，需人工体检确认初始状态 |

**存量机床的根本限制**：建立的基线描述的是"安装传感器时的机床状态"，健康度反映"相对于那个时刻的变化"，而非"相对于出厂新机的健康程度"。这一定位必须在产品层面向用户说明清楚。

### 7.4 基线版本管理

```
基线触发重建的条件：
  大修或换件后（换轴承、换丝杠、重新对中）
  每次大修后强制重建，保留历史版本用于跨版本趋势对比

模型版本命名建议：
  baseline_v1_20240101  出厂/装机后建立
  baseline_v2_20240615  第一次大修后重建
  baseline_vN_YYYYMMDD  每次重大维护后更新
```

### 7.5 健康度的两种语义

```
绝对健康度：相对于出厂初始状态
  需要新机基线作为参照
  存量机床通常无法获得
  适合设备选型、采购决策参考

相对健康度：相对于某个已知基准点（大修后、安装传感器时）
  所有机床都能建立
  适合趋势监测、预测性维护
```

两种健康度在界面上必须区分标注，避免用户错误解读数值含义。

## 10. 第二阶段预留接口

第二阶段在异常检测基础上增加故障定位能力，以下接口在第一阶段设计时即预留，避免后期重构：

**T² 特征贡献输出**：每次告警时输出各原始特征的贡献分值，第二阶段在此基础上增加故障-特征映射规则库。

**原始信号归档**：边缘侧存储每次热机的完整原始信号（时间戳+幅值），不只是14个标量特征，为第二阶段的位置映射和频率分析提供数据基础。存储策略：正常情况保留最近30次，健康度告警时永久保留。

**工况分层标签透传**：工况分组标签（转速档、进给档、班次时长档）随每条数据一起上传，第二阶段可以在特定工况下做更精细的分析。

**模型接口标准化**：健康度输出接口定义为 `(health_score, T2_value, T2_ucl, feature_contributions, confidence_level, baseline_version)`，底层算法从分位数升级到 T² 时上层监测和告警逻辑不变。

## 11. 关键设计决策汇总

| 决策点 | 选择 | 理由 |
|--------|------|------|
| 采集窗口 | 热机后空运行 | 消除刀具/工艺/材料变量，工况可控 |
| 温度处理 | 协变量线性回归去除 | 不建热变形模型，只消除温度对基线的混淆影响 |
| 工况分层 | 转速×进给硬分层 + 温度软回归 | 保留工程语义，避免样本碎片化 |
| 冷启动策略 | 稳定性加权 + 渐进三阶段 | 放弃"磨合结束时刻"判定，用连续权重替代二元切换 |
| 跨机迁移 | 只迁移结构知识，不迁移数值参数 | 电机/轴承型号和装配精度差异导致绝对均值不可比 |
| 特征设计 | 14个时域标量 | 边缘侧轻量计算，无FFT，无坐标对齐，第一阶段只做异常检测 |
| 主轴振动融合 | 冷启动合振幅值，成熟期拆分 | 冷启动特征少基线稳，成熟期捕捉方向性故障 |
| 位置相关特征 | 第一阶段全部排除 | 需坐标对齐或高精度时间同步，超出边缘侧资源限制 |
| 存量机床 | 区分绝对/相对健康度，大修后重建基线 | 无法假设存量机床当前状态为健康状态 |
| 训练部署 | 训练在云端，推理在边缘 | 边缘侧只存 <5KB 参数文件，推理为纯向量运算 |
| 健康度连续性 | 阶段过渡期加权混合（50~200条） | 避免算法切换时健康度跳变 |
| 第一/二阶段分离 | 异常检测与故障定位完全解耦 | 先做扎实再做精细，接口预留避免重构 |
