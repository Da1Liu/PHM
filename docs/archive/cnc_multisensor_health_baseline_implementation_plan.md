# 机床多传感器健康基线落地路线

> 目标：把已验证的“多传感器协方差/PCA 健康基线”路线落到机床进给系统、主轴系统、液压系统。
>
> 当前实验依据：
> - `step7_multisensor_covariance_baseline.py`：多传感器特征自检与真实故障模型对照。
> - `step8_multisensor_weak_feature_ablation.py`：剔除强单变量后的协方差/PCA 对照。
> - `step9_multisensor_relationship_anomaly.py`：边际正常但传感器关系异常的检出测试。
>
> 阶段结论：单变量阈值和对角协方差不足以覆盖多传感器关系异常；工程优先级应为 `PCA + T2 + SPE`，其次为正则化协方差马氏距离，不建议裸 `full covariance + pinv`。

---

## 1. 总体技术路线

### 1.1 核心思想

每次标准采集生成一条多传感器样本：

```text
x_t = [
  vibration_features,
  current_or_power_features,
  temperature_features,
  pressure_or_flow_features,
  position_or_error_features,
  ...
]
```

健康基线学习的是这些变量在健康状态下的联合结构：

```text
健康状态 = 均值向量 mu + 协方差/主成分结构
异常状态 = 幅值偏移 或 多传感器关系偏移
```

推荐评分：

```text
score = max(T2 / T2_UCL, SPE / SPE_UCL)
health = exp(-alpha * score)
```

其中：

- `T2`：主成分空间内异常，反映整体共变模式偏移。
- `SPE`：残差空间异常，反映局部传感器或未建模关系异常。
- `UCL`：由健康校准集分位数给出，建议 99% 或 99.5%。

### 1.2 为什么不用裸完整协方差

实验中 14 维多传感器协方差矩阵条件数达到 `1.1e8` 量级，直接求逆风险高：

```text
裸 full covariance + pinv 可用于实验对照；
不建议作为工程上线算法。
```

工程优先级：

1. `PCA + T2 + SPE`
2. 正则化协方差马氏距离
3. 对角协方差/单变量阈值作为兜底和解释参考

## 2. 数据对齐原则

### 2.1 不建议逐点硬对齐

真实机床中，振动可能是 kHz 级，电流/功率可能是 Hz 到 kHz，温度通常更低频。第一阶段不建议强行逐点拼接：

```text
错误方向：每个振动采样点都找一个电流/温度点拼成向量
```

原因：

- 采样率不同。
- 传感器链路延迟不同。
- 温度动态慢，逐点对齐没有物理意义。
- 振动高频包含周期/冲击信息，电流更多反映负载和摩擦。

### 2.2 推荐窗口级/阶段级对齐

第一阶段采用：

```text
同一标准采集窗口
  -> 截取同一运动阶段或稳态阶段
  -> 各传感器按自身采样率提取统计特征
  -> 拼成一条 cycle-level / segment-level 健康样本
```

这对应 UCI Hydraulic 的验证方式：每个 60 秒标准循环转成一条多传感器特征向量。

### 2.3 什么时候需要严格时间对齐

以下场景才需要更精细的时间对齐：

- 要分析电流突变后振动响应的延迟。
- 要做换向瞬态、启停瞬态、冲击响应建模。
- 要构建序列模型而不是窗口统计健康基线。
- 要定位故障传播链路，而不只是异常检测。

第二阶段可引入：

```text
统一时间戳
重采样
互相关估计延迟
事件触发对齐
动态时间规整
```

但第一阶段健康基线不依赖这些复杂步骤。

## 3. 进给系统方案

### 3.1 标准采集程序

建议每个轴单独采集，每次只运动一个轴：

```text
X/Y/Z 轴分别执行标准往复程序
固定速度档：低速 / 中速 / 高速
固定加速度参数
固定行程区间
每个方向至少 5-10 次往复
```

每次采集分段：

```text
加速段：可保留作瞬态扩展，第一版不进主模型
匀速正向段：主建模段
换向段：第一版剔除
匀速反向段：单独建模或作为方向标签
```

### 3.2 推荐传感器

| 类型 | 信号 | 作用 |
|---|---|---|
| 驱动 | 伺服电流/转矩 | 摩擦、预紧、负载变化 |
| 运动 | 位置误差/跟随误差 | 机械间隙、控制性能 |
| 振动 | 轴承/丝杠/导轨振动 | 局部冲击、松动、磨损 |
| 温度 | 电机/丝杠/环境温度 | 热状态与润滑粘度代理 |

### 3.3 第一版健康向量

按“轴 + 方向 + 速度档”分别建基线：

```text
[
  current_mean,
  current_rms,
  current_std,
  positive_negative_current_diff,
  following_error_rms,
  following_error_p95,
  vibration_rms,
  vibration_kurtosis,
  vibration_p2p,
  motor_temp_mean,
  ambient_temp_mean
]
```

### 3.4 适用性判断

适用，且很有价值。

原因：

- 电流、跟随误差、振动之间存在明确机械耦合。
- 很多早期异常不是单个变量越界，而是“同样速度下电流略升、振动略升、误差略增”的联合偏移。
- 多传感器协方差/PCA 能比单阈值更早捕捉组合退化。

主要风险：

- 工况分层必须严格，否则速度/方向差异会主导协方差。
- 润滑状态和温度会改变电流基线。
- 反向间隙和方向性磨损要求正反向分开建模。

### 3.5 推荐算法

```text
冷启动期：单变量分位数 + CUSUM
成熟期：PCA + T2 + SPE
解释层：SPE/载荷映射回原始特征贡献
```

## 4. 主轴系统方案

### 4.1 标准采集程序

建议使用无切削、固定转速阶梯：

```text
暖机完成后采集
rpm 档位：低 / 中 / 高，例如 1000 / 3000 / 6000 rpm
每档稳态保持 20-60 秒
每档单独形成一条或多条样本
```

不要把不同转速混在同一个基线中。

```text
baseline_key = spindle_id + rpm_bin + thermal_state
```

### 4.2 推荐传感器

| 类型 | 信号 | 作用 |
|---|---|---|
| 振动 | 前/后轴承座径向振动 | 轴承状态、动平衡、松动 |
| 驱动 | 主轴电流/功率 | 空载损耗、摩擦、负载代理 |
| 温度 | 前/后轴承温度、电机温度 | 热稳定、轴承摩擦 |
| 转速 | 实际 rpm | 工况分层和同步参考 |

### 4.3 第一版健康向量

每个 rpm 档单独建模：

```text
[
  vib_x_rms,
  vib_y_rms,
  vib_resultant_rms,
  vib_kurtosis,
  vib_crest,
  spindle_current_mean,
  spindle_current_std,
  spindle_power_mean,
  front_bearing_temp_mean,
  rear_bearing_temp_mean,
  motor_temp_mean,
  temp_delta_front_rear
]
```

### 4.4 适用性判断

适用，但比液压和进给更挑剔。

适用点：

- 主轴轴承温升、振动、功率之间存在明确耦合。
- 动平衡变化、轴承损伤、润滑劣化常表现为多变量联合偏移。
- 单变量阈值容易漏掉“都略有变化但单项未越界”的早期状态。

关键风险：

- rpm 对振动幅值影响极强，必须按转速分层。
- 共振区会导致协方差结构突变，不能混入普通稳态基线。
- 刀柄/夹具/安装状态会改变振动基线；标准采集应无刀或固定标准刀柄。
- 冷机到热稳态期间，温度和振动关系随时间变化，建议分热态阶段。

### 4.5 推荐算法

```text
每个 rpm 档：PCA + T2 + SPE
跨 rpm：不要直接共享协方差矩阵，可共享特征体系和阈值策略
温度处理：第一版进健康向量；第二版再做温度残差对照
```

## 5. 液压系统方案

### 5.1 标准采集程序

液压系统最接近 UCI Hydraulic 验证场景。建议设计固定负载循环：

```text
固定建压动作
固定保压时间
固定卸压动作
固定油温或记录油温
每次循环 30-120 秒
```

每个循环提取一条多传感器样本。

### 5.2 推荐传感器

| 类型 | 信号 | 作用 |
|---|---|---|
| 压力 | 主/支路压力 | 泵、阀、泄漏、蓄能器状态 |
| 流量 | 主/回油流量 | 内泄漏、效率 |
| 功率 | 泵电机电流/功率 | 负载与效率 |
| 温度 | 油温、环境温度 | 粘度、热状态 |
| 振动 | 泵体振动 | 机械冲击、气蚀、轴承 |

### 5.3 第一版健康向量

```text
[
  pressure_mean,
  pressure_std,
  pressure_p95,
  flow_mean,
  flow_std,
  q_over_p,
  motor_power_mean,
  motor_power_std,
  oil_temp_mean,
  oil_temp_slope,
  pump_vibration_rms,
  pump_vibration_kurtosis
]
```

### 5.4 适用性判断

非常适用。

原因：

- 压力、流量、功率、油温之间存在强物理耦合。
- 故障经常表现为效率下降和关系改变，而不是单点突变。
- UCI 实验已经证明：剔除强单变量后，PCA+T2+SPE 和正则化协方差仍明显优于对角模型。

主要风险：

- 油温强烈影响粘度，进而影响压力/流量/功率关系。
- 负载循环必须标准化，否则负载差异会主导健康分布。
- 若阀动作阶段与稳态保压段混合，协方差结构会变宽。

### 5.5 推荐算法

```text
第一版：PCA + T2 + SPE
温度：先进入健康向量；后续做温度残差对照
阈值：健康校准集 99% 分位数
告警：EWMA(score) + 连续 K 次超限
```

## 6. 基线分层策略

### 6.1 必须分层的变量

| 系统 | 必须分层 |
|---|---|
| 进给 | 轴、方向、速度档、运动阶段 |
| 主轴 | rpm 档、热态阶段、刀柄/无刀状态 |
| 液压 | 负载循环类型、保压/建压/卸压阶段、油温区间 |

### 6.2 不建议混合建模的情况

```text
不同转速混一个主轴基线
进给正反向混一个基线
液压建压瞬态和保压稳态混一个基线
冷机和热稳态混一个基线
维修前后混一个基线
```

这些混合会让协方差矩阵学习到“工况差异”，而不是健康状态。

## 7. 冷启动和成熟期切换

### 7.1 冷启动期

样本少时不适合估计高维协方差：

```text
n < 30:
  单变量分位数
  工程阈值
  CUSUM 相对变化
```

### 7.2 过渡期

```text
30 <= n < 100:
  对角协方差
  正则化协方差
  特征数量限制在 5-8
```

### 7.3 成熟期

```text
n >= 100:
  PCA + T2 + SPE
  或正则化协方差
```

建议至少满足：

```text
n_train >= 10 * feature_count
```

否则必须降维或正则化。

## 8. 告警与解释

### 8.1 告警逻辑

不建议单点超限立刻报警：

```text
raw_score = max(T2/T2_UCL, SPE/SPE_UCL)
ewma_score_t = lambda * raw_score_t + (1 - lambda) * ewma_score_{t-1}
alarm = ewma_score > threshold 持续 K 次
```

建议：

```text
lambda = 0.1-0.2
K = 3-5
```

### 8.2 贡献解释

PCA/T2/SPE 告警后输出：

```text
top contributing features
T2 contribution
SPE residual contribution
baseline version
current operating layer
```

解释例子：

```text
主轴：front_bearing_temp + vib_resultant_rms + power_mean 同时偏离
进给：current_rms + following_error_rms + vibration_kurtosis 同时偏离
液压：flow_mean + q_over_p + motor_power_mean 同时偏离
```

## 9. 实验对真实机床的外推边界

已完成 UCI 实验能证明：

- 多传感器联合基线可行。
- 去掉强单变量后，协方差/PCA 仍有收益。
- 边际正常但关系异常时，对角模型基本失效，PCA+T2+SPE 和正则化协方差有明显优势。

不能直接证明：

- 某台真实机床上的长期误报率。
- 所有主轴/进给/液压结构都共用一套数值阈值。
- 未经工况分层的数据可以直接建模。
- 时空未对齐问题能由协方差模型自动解决。

真实部署必须补充：

```text
标准采集程序验证
传感器同步/窗口对齐验证
工况分层验证
维修事件后的基线重建策略
现场误报率监控
```

## 10. 下一步建议

### 10.1 代码验证层

把当前 UCI 验证脚本整理为统一 pipeline：

```text
feature extraction
baseline split
PCA + T2 + SPE training
threshold calibration
EWMA alert
feature contribution
report generation
```

### 10.2 机床试采层

优先选一个系统做试采，建议顺序：

1. 液压系统：最容易标准化，最接近已验证数据。
2. 主轴系统：价值高，但 rpm/热态分层要求严格。
3. 进给系统：需要处理轴、方向、速度、运动阶段分层。

### 10.3 最小现场试验

每个分层至少采：

```text
健康样本 100 条以上
跨 2-4 周
覆盖不同温度/班次
每次采集保留原始信号和特征
```

第一版验收：

```text
健康段 FAR < 5%
EWMA 无连续误报
人工制造或历史异常能触发 score 升高
贡献特征可解释
```
