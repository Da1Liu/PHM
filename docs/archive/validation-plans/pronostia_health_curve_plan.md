# PRONOSTIA 健康曲线验证方案（最小立柱版）

> 目标：验证「健康曲线能不能做成」这件事本身——在**单一轴承、单一工况**下，证明从原始振动出发的健康基线技术路线，能产出一条对真实退化单调敏感、且能提前告警的健康曲线。
> 本步**刻意不做**多工况响应面归一化、不做温度补偿、不做多轴承泛化。那些是叠在本层之上的精化，先验证底座。
> 本文档只定义实验设计、判据与停下点，不写代码。

---

## 0. 这一步在整体技术路线里的位置

完整技术路线（`cnc_health_baseline_technical_plan.md`）的核心创新是**多工况协变量回归归一化**——把转速/进给/温度对健康敏感信号的影响回归掉，残差只对退化敏感。

但那层精化的前提，是底层链路先成立：

```
原始振动 → 特征提取 → 健康基线 → 异常评分 → 健康曲线 → 趋势告警
```

PRONOSTIA 的单轴承 run-to-failure 是验证这条底座的**理想环境**：单个轴承全程工况恒定，**没有任何需要归一化的协变量**，信号变化纯粹来自退化。所以本步把技术路线剥到只剩底座，单独验证它。归一化层留到后续用 Paderborn（多工况健康基线）验证。

## 2. 选定立柱轴承：Bearing1_1

理由：

- 寿命最长（2803 快照），退化过程采样最充分，健康曲线分辨率最高。
- PHM2012 文献中被研究最多，有公开 baseline 可对照（曲线形状、告警时机是否落在公认区间）。
- 有温度文件（备用，本步不使用）。

判定通过后是否换/加轴承，由用户在停下点决定（见 §7）。

## 3. 关键假设（必须显式声明）

本步结论强依赖以下假设，任何一条变动都会影响结论，故全部显式列出：

1. **无健康度真值**：PRONOSTIA 只有快照时间序号和最终失效点，没有逐点健康标签。因此「曲线跟踪退化」只能用**距失效时间**作代理来衡量，验证的是**单调性 + 告警提前量**，不是绝对健康值的准确性。

2. **健康基线窗口靠假设划定**：取寿命**前若干比例**的快照为「健康」来建基线。第一版取**前 10%**（约 280 个快照），并做敏感性检查（见 §6）。

3. **本步为方法论验证，非目标域验证**：FEMTO 是滚动轴承，结论证明的是「这套链路能产出可用健康曲线」这个方法论命题，**不直接证明**迁移到液压/主轴/进给系统的适用性。

## 4. 链路设计（按用户决策：单轴承 + 单一指标起步）

### 4.1 特征提取

每个 acc 快照 → 一组振动特征。第一版选对滚动轴承退化公认敏感的时域特征（信号优先用水平方向，垂直方向作对照）：

| 特征 | 退化敏感性 |
|------|-----------|
| RMS | 能量整体上升 |
| 峭度 Kurtosis | 早期冲击敏感 |
| 峰值因子 Crest factor | 冲击/能量比 |
| 峰峰值 Peak-to-peak | 幅值包络 |

输出：一条 `(快照序号, 特征向量)` 时间序列。先画特征-时间原始曲线自检，确认至少有特征随退化呈上升趋势。

### 4.2 健康基线

- 取前 10% 快照为健康样本，在其上估计每个特征的均值与协方差。
- 标准化（用健康段的 mean/std）。

### 4.3 评分引擎（用户选定：先单一指标）

用**单一健康基线的马氏距离 / Hotelling T²**：

```text
T²(t) = (x_t − μ_health)ᵀ Σ_health⁻¹ (x_t − μ_health)
```

- 这是「单一指标」最自然的实现：一个数，越大越偏离健康。
- 健康度映射：`health(t) = exp(−α · T²(t) / T²_baseline_ref)`，α 待定、做几组对比。
- 控制限 UCL：用健康段 T² 的高分位数（如 99%）或 F 分布理论值，两者并列输出。

**刻意不在本步做**：PCA 降维、SPE 残差统计、冷启动分阶段切换。这些是用户明确选定推迟的复杂度，跑通单一指标后再按停下点决定是否加。

### 4.4 趋势告警

对 T²（或 health）做 EWMA：

```text
ewma_t = λ·score_t + (1−λ)·ewma_{t−1}
```

- 对比 λ = 0.1 / 0.15 / 0.2。
- 告警条件：ewma 越过基于健康段标定的阈值。
- 输出首次告警快照序号。

## 5. 评价指标（最小集）

| 指标 | 目的 | 通过判据 |
|------|------|----------|
| 健康段误报率 | 健康期是否稳 | < 5% |
| Spearman(health, 距失效时间) | 退化段单调性 | 显著正相关（health 随接近失效而下降） |
| 首次告警提前量 | 告警是否及时 | 在失效前给出告警，且提前量落在合理区间 |
| 曲线形状 vs 文献 | 合理性旁证 | Bearing1_1 曲线形状与公开研究无明显矛盾 |

---

## 6. 敏感性检查（防止结论被假设绑架）

- **健康窗口比例**：前 5% / 10% / 20% 三档，看健康曲线和告警时机是否稳定。若剧烈变化，说明结论对该假设过敏，需在结论中标注。
- **特征子集**：全 4 特征 vs 仅 RMS+峭度，看是否某个特征主导。
- **方向**：水平 vs 垂直加速度，看结论是否依赖单一通道。

---

## 7. 停下点（每个关键节点交用户决策）

按用户要求，以下节点**停下来给结果、由用户定方向**，不自动推进：

1. **特征自检后**：展示 4 个特征的时间曲线。确认「有特征随退化上升」后，再决定进不进评分。若特征对退化不敏感，停下重选特征。
2. **健康曲线出来后**：展示 Bearing1_1 的 health/T² 曲线 + EWMA + 告警点 + 指标表。这是本步的核心交付。由用户判断「健康曲线做成了没有」。
3. **判定通过后**：由用户决定下一步方向——
   - 是否泛化到多轴承（标准版）
   - 是否加 PCA+SPE / 冷启动分阶段（补全成熟期引擎）
   - 是否转去 Paderborn 验证多工况响应面归一化（你的核心创新）

---

## 8. 本步能证明 / 不能证明

**能证明**：

- 从原始振动能否提取对退化敏感的特征。
- 用寿命早期建的单一基线，健康期能否保持低误报。
- 单一指标（T²/马氏距离）的健康曲线能否随真实退化单调下降。
- EWMA 能否在失效前以合理提前量告警。

**不能证明**（按设计推迟）：

- 多工况响应面归一化——你的核心创新，本步单工况，不碰。
- 温度补偿——工况恒定，温度仅小幅漂移。
- 跨轴承/跨工况泛化——本步单轴承。
- 迁移到液压/主轴/进给的目标域适用性——FEMTO 是滚动轴承，本步是方法论验证。

---

## 9. 已知风险

- FEMTO 部分轴承为「长期平稳→临失效突跳」的断崖式退化。对异常检测其实合适（要的就是抓住突跳），但 Spearman 单调性可能因前段过平而偏低，需结合告警提前量综合判断，不单看相关系数。
- 健康度无真值，所有「准确性」类结论都受限于「距失效时间」这个代理，须在结论中明确。
