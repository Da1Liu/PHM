# ADR-002 温度分混淆 / 耦合两类角色

状态: 已采纳 (step5 vs step9)

## Decision
温度信号按角色分两类, 处理方式相反, 落到 `signal.temp_role`:
- **混淆温度** (环境 / 电机) → `confound` → 回归剔除 (`TempResidualizer`)。
- **耦合温度** (轴承 / 油温) → `coupled` → 进特征向量。

## Reason
- step5: 混淆温度回归剔除使单通道检测预警提前量 194→407min (剔除温度引入的虚假漂移)。
- 耦合温度本身携带退化信息 (轴承发热 = 故障征兆), 剔除会丢信息。

## Consequence
- 主轴**热态作协变量 (残差化), 非分层** —— 效应平滑可残差化, 且"热机后采"协议已标准化热态 → 区别于 rpm 强非线性必须分层 (见 [[ADR-003]])。
- 残差化"基线集拟合后冻结", 在线打分用冻结系数 (`covariate.py` / `engine.py`)。
- 别把两类混为一谈: 错把耦合温剔除 = 丢故障信号; 错把混淆温进向量 = 引虚假漂移。
