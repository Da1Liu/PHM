# ADR-003 工况分层 = 诊断分辨率旋钮, 取最粗分层

状态: 已采纳 (原则已定, 粒度待现场标定)

## Decision
- **同工况内才能比**; 分层粒度取"满足诊断需求的最粗分层", 非外部强制。
- 已定: 液压 **单基线** / 主轴 **rpm档** (热态=协变量, 不分层) / 进给 **轴×方向×速度档**。

## Reason
- UCI 实测: 把混工况样本当一个健康池 → 训练/打分落不同工况 → **FAR 可达 93%**。
- 健康数据不稀缺 (用户纠正), 故分层粒度可自定, 不必为稀缺让步。
- **rpm 强非线性 → 必须分层**; **热态平滑 → 残差化作协变量** (见 [[ADR-002]])。两者别混。

## Consequence
- regime 由 NC 空跑程序定义 (单一定义源, `nc_profile.py` → `ACQUISITION_CONTRACT.md §6`); 档边界 = 相邻设定点中点, 用设定值非实测值分箱。
- NC 程序扫 rpm 各箱**并行填充**, 分层不再是日历瓶颈。
- 稳态门控两层 (程序标记权威 + 信号 CV 兜底); 非稳态不准入基线。
- 门控阈值 / rpm 档边界 / 进给档粒度待首台真实数据标定 (阶段 E)。
