# ADR-004 自基线 + 大修 epoch reset, 跨 reset 不可比

状态: 已采纳

## Decision
- **自基线**: 机床与自己的过去比 (不与同型号参考机比)。
- 大修 / 拆装后 `machine.current_epoch` +1; 综合健康分**只在一个 epoch 内有效**。

## Reason
- 部署现实: **无同型号健康参考机**; 系统既上新机也上**存量设备**。现成方案 (要参考机/故障样本) 给不了。
- 大修会阶跃改变机床状态, 跨 reset 的绝对健康分不可比。

## Consequence
- 维护好坏**不看综合健康分**, 看绝对裸指标台阶 + 维护后稳定性。
- 外部锚点 = 存量设备客观退化 (一直加工) + 人工大修 reset + 用户确认精度是否达标。
- telemetry / health_result 全表带 `epoch`; 评分按 (machine, system, epoch) 独立。
- 存量策略: 先验证算法能否揭示存量机退化现象, 通过后靠人工大修 reset 正式起基线。
